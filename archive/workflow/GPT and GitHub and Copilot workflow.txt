    Worth keeping

    A document

    A design decision

    A checklist

    A theory version

    A workflow description

Step 3: Store It in GitHub

When you put something in GitHub, you are saying:

    You can safely revise without losing history

    You can return to any earlier state

Now Copilot becomes useful.

    Draft text you already decided belongs there

    Be trusted without review

    Copilot writes for you, not instead of you.

Once something exists in GitHub:

Bring it back to ChatGPT and ask:

    “Is this internally consistent?”

    “What’s missing?”

    “What assumptions am I making?”

    “What would a skeptic say?”

    Make deliberate changes

    Commit them to GitHub

    Preserve the history
